MODULE 1

Networking & Device Identity

Understanding the Digital Landscape

1. OSI Model (The Framework) - Slide 1

  • Definition: A 7-layer map showing how data travels from wires to apps.
  • Full Form: Open Systems Interconnection Model.
  • Physical Layer (L1): The actual cables/signals; first place to check if hardware is tampered.
  • Network Layer (L3): Where IP Addresses live; identifies the "digital house."
  • Application Layer (L7): Where users interact (WhatsApp/Email); contains the actual content.
Dhenkanal Police Training

1. OSI Model (The Framework) - Slide 2

  • Investigation Use: Helps narrow down if a crime was a network hack or an app bypass.
  • Data Link (L2): Contains the MAC address, identifying the physical hardware.
  • Reading logs: Technical logs usually follow these layers; read from bottom up.
  • Where found: Conceptual framework used in all network server and router logs.
  • Police Tip: Most cybercrime evidence is found at Layer 3 (IP) and Layer 7 (App).
Dhenkanal Police Training

2. IP Addresses (The Address) - Slide 1

  • What it is: A unique numeric label assigned to every device on a network.
  • Full Form: Internet Protocol Address.
  • IPv4: Four sets of numbers (e.g., 103.4.12.1); most commonly seen in India.
  • IPv6: Long alphanumeric strings; used as we run out of IPv4 addresses.
  • Public IP: The identity of the entire home/office visible to the global web.
Dhenkanal Police Training

2. IP Addresses (The Address) - Slide 2

  • Private IP: Internal address (e.g., 192.168.x.x) given by a local router.
  • Investigation Use: The first step to identifying the internet connection used in a crime.
  • Where found: Server logs, email headers, and social media login history.
  • Reading it: The first half usually identifies the ISP (Internet Service Provider).
  • Crucial Rule: An IP address is not a person; it is a connection point.
Dhenkanal Police Training

3. Static vs Dynamic IPs - Slide 1

  • Static IP: An address that never changes (like a permanent home address).
  • Dynamic IP: An address that changes every few hours/days.
  • Why it matters: Most Indian mobile users (Jio/Airtel) use Dynamic IPs.
  • Investigation: You must provide an exact timestamp to ISPs to get the correct user.
  • Lease Time: The duration an IP is assigned to one phone before it moves to another.
Dhenkanal Police Training

3. Static vs Dynamic IPs - Slide 2

  • ISP Assignment: ISPs use a pool of IPs to serve millions of customers.
  • DHCP Server: The computer at the ISP that rotates these IPs automatically.
  • Radius Logs: The specific report from ISPs showing who had a Dynamic IP at a specific time.
  • Reading: Look for "Allocation Time" and "Release Time" in ISP reports.
  • Error Warning: A 1-minute time error in your request can lead to the wrong suspect.
Dhenkanal Police Training

4. MAC Address (The Hardware ID) - Slide 1

  • Full Form: Media Access Control.
  • What it is: A permanent "fingerprint" burned into the device's Wi-Fi/Bluetooth chip.
  • Format: Six pairs of hex characters (e.g., 00:1A:2B:3C:4D:5E).
  • Manufacturer ID: The first 3 pairs identify the brand (Apple, Samsung, etc.).
  • Investigation Use: Proves a specific physical device was used, regardless of the SIM.
Dhenkanal Police Training

4. MAC Address (The Hardware ID) - Slide 2

  • Where found: On a sticker on the device or in "About Phone" settings.
  • Investigation Trace: Found in the logs of a router the suspect connected to.
  • Spoofing: Technical users can "fake" it, but the physical ID remains.
  • Permanence: Unlike IP addresses, MAC addresses do not change with the network.
  • Smoking Gun: Matching a MAC address from a crime-scene router to a seized phone.

MODULE 2

Telecom & Data Records

Working with Service Provider Data

1. CDR (Call Detail Record) - Slide 1

  • Full Form: Call Detail Record.
  • Definition: Digital log of all calls/SMS made or received by a number.
  • Investigation Use: Establishes relationship between suspects via call frequency.
  • Data Included: Caller, Receiver, Date, Time, Duration, and Cell Tower ID.
  • What's NOT inside: CDRs do not contain audio recordings or SMS text.
Dhenkanal Police Training

1. CDR (Call Detail Record) - Slide 2

  • SMS Logging: Shows when an SMS was sent but lists duration as 0.
  • Tower ID: Every call record shows the tower location used at that moment.
  • IMEI Link: Shows which physical phone was used to make the call.
  • How to read: A-Party (Caller), B-Party (Called), Cell ID (Location).
  • Analysis: Used to find "Common Numbers" between two different crime scenes.
Dhenkanal Police Training

2. IPDR (Internet Logs) - Slide 1

  • Full Form: Internet Protocol Detail Record.
  • Definition: A log of when a mobile user connected to the internet.
  • Investigation Use: Maps a "Crime IP" back to a specific Phone Number.
  • Retention: Indian ISPs keep these logs for at least 2 years.
  • Data Type: Shows Public IP, Private IP, and Login/Logout timestamps.
Dhenkanal Police Training

2. IPDR (Internet Logs) - Slide 2

  • Session Duration: Shows how long the suspect stayed online.
  • Destination IP: Sometimes shows which website (e.g., Facebook) was visited.
  • Volume: Shows how much data was uploaded (sending threat) vs downloaded.
  • MAC Address: Often included to identify the specific handset on the data network.
  • Key Rule: Use this to prove a suspect was online at the time of a cyber-attack.

MODULE 3

Web & Domain Fundamentals

Investigating Fraudulent Websites

1. DNS (Domain Name System) - Slide 1

  • Full Form: Domain Name System.
  • Definition: The "Phonebook" turning names (google.com) into IPs.
  • A-Record: The specific record mapping a site name to its server IP.
  • Investigation Use: Tells you where the physical server of a scam site is located.
  • Where found: Publicly via "DNS Lookup" tools.
Dhenkanal Police Training

1. DNS (Domain Name System) - Slide 2

  • MX-Record: Shows the server used by the suspect to send emails.
  • NS-Record: Name Servers; identify which company manages the site traffic.
  • DNS Cache: Can reveal old IPs even after a criminal moves their site.
  • Analogy: DNS is the GPS coordinate for a house name.
  • How to read: [Domain] -> [Points To] -> [IP Address].
Dhenkanal Police Training

2. WHOIS Lookup - Slide 1

  • Definition: A query to find who registered a specific website.
  • Investigation Use: Finding the name, email, and phone of a fraudster.
  • Privacy Guard: Note that criminals often use services to hide their real data.
  • Creation Date: A bank site created only 1 day ago is a 100% fake.
  • Location: available at whois.com or centralops.net.
Dhenkanal Police Training

MODULE 4

Social Media & OSINT

Open Source Intelligence & Online Evidence

1. Facebook/Instagram Portals - Slide 1

  • What it is: A dedicated portal for Police to request data from Meta.
  • Requirements: Requires an official **.gov.in** or **.nic.in** email ID.
  • Preservation: You can request a 90-day "Freeze" on a suspect's data instantly.
  • Emergency: Used for life-safety cases like kidnappings to get data in minutes.
  • Location: facebook.com/records.
Dhenkanal Police Training

1. Facebook/Instagram Portals - Slide 2

  • Data Provided: IP logs, login history, and registration phone number.
  • BSI: Basic Subscriber Information—the first document you should request.
  • Legal: You must upload your Section 91 (BNSS 94) notice as a PDF.
  • Note: Meta will not share data via random Gmail requests.
  • Evidence: Maps a social media profile to a physical IP and Phone number.
Dhenkanal Police Training

2. Reverse Image Search - Slide 1

  • Definition: Using a photo to find where else it appears on the web.
  • Tools: Google Images, Yandex, and PimEyes.
  • Investigation Use: Checking if a suspect's DP is stolen from a model.
  • Fake Clue: If a "Girl's Profile" uses a stock photo, it is likely a honey-trap.
  • Location: images.google.com.
Dhenkanal Police Training

2. Reverse Image Search - Slide 2

  • Original Source: Helps find the first time a photo was posted (The Real Owner).
  • Identifying Suspects: Upload a CCTV frame to find the suspect's real name.
  • Building Match: Search for background buildings to identify a suspect's city.
  • Yandex: Particularly powerful for finding faces in India/Asia.
  • Police Tip: Use this to verify identity before making a physical arrest.
