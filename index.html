OSI Model — "How Data Travels" Map
Full Form
Open Systems Interconnection Model.
What it is
A 7-layer framework describing how a message moves from a human tapping a screen to a signal traveling through a wire.
Why it's needed
Helps police identify where a crime occurred — was it a physical break-in (Layer 1) or a hacked WhatsApp account (Layer 7)?
General Location
A conceptual "rulebook" used by every computer in the world.
Investigation Location
When you seize a device, look at Layer 7 logs — Application logs like WhatsApp, Instagram.

OSI Model — "How Data Travels" Map
How to read it
Layer 1 is the "Ground" (Cables) and Layer 7 is the "Top" (The App).
Data Link — Layer 2
This is where the MAC Address lives; it connects the hardware to the local network.
Network — Layer 3
This is where the IP Address lives; it helps data find the right "digital house."
Transport — Layer 4
Ensures the message arrives in the right order and isn't broken.
The Analogy
Think of it like a courier service. Layer 1 is the road, Layer 3 is the address on the box, and Layer 7 is the letter inside.

IP Address — "Digital House Number"
Full Form
Internet Protocol Address.
What it is
A unique string of numbers assigned to every device on a network.
Why it's needed
It is the first clue in any cybercrime. It tells you which internet connection was used to commit the act.
General Location
Found in your phone's "Network Settings" or by searching "What is my IP" on Google.
Investigation Location
Found in the server logs of the victim — e.g. the IP address that logged into a stolen bank account.

IP Address — "Digital House Number"
IPv4 Appearance
Four sets of numbers separated by dots — e.g. 103.45.2.11.
IPv6 Appearance
Eight groups of letters and numbers — e.g. 2001:db8:0:0:0:ff00:42:8329.
Public IP
The address the whole world sees, assigned by the ISP.
Private IP
The address used inside a home/office, assigned by the local router.
Key Insight
A Public IP identifies the house/office, while a Private IP identifies the specific device inside that house.

Static vs. Dynamic IPs — "The Changing Address"
What it is
Static means the IP never changes. Dynamic means the IP changes every time the user restarts their phone or router.
Why it's needed
To understand why an IP address you found yesterday might belong to a different person today.
The Indian Context
Most mobile users in India (JIO, Airtel) use Dynamic IPs to save resources.
Investigation Challenge
If you track a Dynamic IP without a timestamp, you might arrest the wrong person.
Static Use Case
Usually used by big companies, banks, or police headquarters so their servers are always easy to find.

Static vs. Dynamic IPs — "The Changing Address"
Lease Time
In Dynamic IPs, a user "rents" an IP for a few hours. This is called a Lease.

Critical Step
Always ask the ISP: "Who was using this IP at exactly 14:05:10 IST on Oct 12?"

MAC Address — "Hardware Fingerprint"
Full Form
Media Access Control Address.
What it is
A permanent, physical ID number burned into the Wi-Fi or Ethernet chip of a device at the factory.
Why it's needed
Even if a suspect changes their IP or SIM card, the MAC address of the laptop/phone remains the same.
General Location
On a sticker at the back of a laptop or under the "About Phone" menu.
Investigation Location
Found in the logs of the Wi-Fi router used by the suspect.

MAC Address — "Hardware Fingerprint"
How it looks
Six pairs of characters — e.g. 00:0A:95:9D:68:16.
NIC
Network Interface Card — the physical part inside the phone that has the MAC address.
Manufacturer Clue
The first three pairs (e.g. 00:0A:95) tell you which company made the device — Apple, Samsung, etc.
Spoofing
Tech-savvy criminals can "fake" a MAC address using software, though the physical one stays the same.
Evidence Value
Linking a MAC address found in a crime scene router to a laptop found in a suspect's house is a "smoking gun."

DHCP Logs — "The Record Book"
Full Form
Dynamic Host Configuration Protocol Logs.
What it is
A digital diary kept by a router or ISP that records which IP was given to which device.
Why it's needed
It is the bridge that connects an IP Address to a MAC Address.
General Location
Inside the memory of a Wi-Fi router or the ISP's database.
Investigation Location
You seize the router from a suspect's house and download these logs.

DHCP Logs — "The Record Book"
How to read it
It looks like a spreadsheet with columns for "Time," "MAC Address," and "Assigned IP."
Reservation
Sometimes a router is set to always give the same IP to the "Boss's Laptop." This is a "DHCP Reservation."
Scope

Lease Duration
Tells you how long the suspect was connected to that specific Wi-Fi.
Legal Use
These logs prove that Device A was present on Network B at Time C.

Subnetting & Gateway — "The Local Map"
Subnetting
Dividing a large network into smaller "neighbourhoods" (subnets).
Gateway
The "Main Gate" or router that connects a local network to the internet.
Why it's needed
To narrow down where a suspect is — are they in the "Accounting Subnet" or the "Guest Wi-Fi Subnet"?
General Location

Investigation Location
Helps you find the physical router (Gateway) that needs to be seized.

Subnetting & Gateway — "The Local Map"
How it looks
A Subnet Mask usually looks like 255.255.255.0.
The Gateway IP
Usually ends in .1 — e.g. 192.168.1.1. This is the router's internal address.
Public Gateway
For an ISP, this is the "Tower" or "Exchange" the user is connected to.
IP Conflict
If two devices have the same IP on one subnet, the internet stops working — a clue that a new device just joined.
Officer's Goal
Find the Gateway IP to find the physical hardware controlling the traffic.

IMEI vs. IMSI — "Phone vs. SIM"
IMEI Full Form
International Mobile Equipment Identity.
IMSI Full Form
International Mobile Subscriber Identity.
IMEI — The Phone
A 15-digit number unique to the handset. It doesn't change if you change the SIM.
IMSI — The SIM
A 15-digit number unique to the SIM card/Subscriber. It travels with the SIM to new phones.
Why it's needed
To track a person even if they switch phones (track IMSI) or switch SIMs (track IMEI).

IMEI vs. IMSI — "Phone vs. SIM"
General Location
IMEI: Dial *#06# or check the box. IMSI: Stored inside the SIM card.
Investigation Location
Found in CDRs (Call Detail Records) provided by companies like JIO/Airtel.
How to read
IMEI usually starts with 35 or 86. IMSI in India usually starts with 404 or 405.
Blacklisting
If a phone is stolen, the police "Blacklist" the IMEI so no SIM card will work in it.
The Link
Every time a call is made, the tower records BOTH IMEI and IMSI. This links a specific person to a specific handset.

Cell Tower Triangulation — "The Search Area"
What it is
Using the signal strength from 3 or more towers to calculate a suspect's location.
Why it's needed
To find a suspect's live location even if they have turned off GPS/Location services.
How it works
The phone is always "talking" to the nearest towers to stay connected.
Accuracy
In cities (many towers), it is very accurate — within 50–100 metres. In villages, it may only be within 2–3 kilometres.
General Location
Only accessible by the Telecom Service Provider (TSP).

Azimuth/Sector

Wi-Fi SSID & BSSID — "The Wireless Clues"
SSID Full Form
Service Set Identifier — the name you see, like "Home_WiFi".
BSSID Full Form
Basic Service Set Identifier — the MAC address of that specific router.
Why it's needed
A suspect can name their Wi-Fi "Police_Station" (SSID) to prank people, but the BSSID will always be a unique serial number.
General Location
Visible on any phone when searching for Wi-Fi.
Investigation Location
Check the "Saved Networks" on a suspect's seized phone — it shows everywhere they have been.

Wi-Fi SSID & BSSID — "The Wireless Clues"
BSSID Mapping
Websites like Wigle.net keep a global map of BSSIDs. You can search a BSSID and see exactly where that router is located.
Public Hotspots
If a crime was committed using "Railway_Free_WiFi," the BSSID tells you exactly which platform the suspect was standing on.
Hidden SSID

RSSI
This is the signal strength. A high RSSI means the suspect was standing very close to the router.
Evidence Value
Proves the suspect's device was connected to a specific router at a specific location and time.

VPN / Proxy Mechanics — "The Mask"
Full Form
Virtual Private Network.
What it is
An app that creates an encrypted "tunnel." It hides your real Indian IP and replaces it with a foreign IP — e.g. USA.
Why it's needed
To understand why a "Threat Mail" appears to come from New York when the suspect is in Mumbai.
Proxy vs. VPN
A Proxy only hides your IP for one app (like a browser). A VPN hides everything on the phone.
The "Middle-Man"
The data goes: Suspect → VPN Server → Internet. The internet only sees the VPN Server's IP.

VPN / Proxy Mechanics — "The Mask"
Logs
Most VPNs claim a "No-Logs" policy, making it very hard for police to get the real IP.
General Location
Look for apps like NordVPN, ExpressVPN, or "Turbo Proxy" on the suspect's phone.
Investigation Clue
If you see an IP address belonging to "Amazon Data Center" or "Google Cloud" instead of a home ISP, it is likely a VPN/Proxy.
Encryption
VPNs scramble the data. Even if you intercept the Wi-Fi signal, you can't read the messages.
Officer's Strategy
If a VPN is used, look for "Physical Evidence" (like a logged-in session on a laptop) rather than trying to track the IP through the foreign VPN company.

DNS — Domain Name System
What it is
The "Phonebook of the Internet." It translates human names (google.com) into computer numbers (142.250.190.46).
Why it's needed
If you have a scam website name, DNS records tell you which IP address it is currently pointing to.
A-Record
The most common record; it maps the domain name directly to an IPv4 address.
CNAME
An alias record. It shows if a URL is actually just a mask for another hidden website.
MX-Record
Shows the "Mail Exchange" server — tells you which service the criminal is using to send emails.

DNS — Domain Name System
NS-Record
"Name Server" records show which company is managing the website's directions — e.g. Cloudflare or GoDaddy.
DNS Cache
Computers "remember" DNS for a while. Even if a criminal deletes a site, the cache might still show the old IP.
General Location
Publicly available through "DNS Lookup" tools online.
Investigation Use
If a site is taken down, you look at "Historical DNS" to see where it used to be hosted.
How to read it
Domain: scam.com → Points to IP: 1.2.3.4.

WHOIS Lookup
What it is
A public database query that asks, "Who owns this domain?"
Why it's needed
To find the name, email, and phone number of the person who bought the website.
Privacy Protection
Many criminals use "WhoisGuard" or "Privacy Protect" to hide their real name in public results.
Data Included
Registration date, Expiry date, and the "Registrar" (the shop where they bought it).
Technical Contact
Often shows the email of the person who set up the site — a major lead for police.

Domain Registrars
What it is
The "Shop" or "Company" where a person buys a domain name — e.g. GoDaddy, Namecheap, BigRock.
Why it's needed
They are the legal entity you send a notice to when you want to suspend/block a scam website.
The Registry
Above the registrar is the Registry (like Verisign for .com). Registrars are the middle-men.
Abuse Contact
Every registrar has an "abuse@" email address specifically for police to report scams.
KYC
Registrars keep credit card or PayPal details of the buyer — this is how you find the money trail.

Web Hosting — Shared vs. VPS
What it is
The "Digital Land" where the website's files — images, code, stolen data — are stored.
Shared Hosting
Like an apartment building; many websites live on one server. Cheap and common for small scams.
VPS (Virtual Private Server)
Like a private townhouse; more control and harder to investigate because the criminal has "Admin" rights.
Why it's needed
The "Hosting Provider" holds the actual evidence — the stolen usernames/passwords.
General Location
Identified by the IP address of the website.

HTTP/HTTPS Headers
What it is
Hidden "metadata" sent between a browser and a server during every visit.
User-Agent
A header that reveals the suspect's Browser (Chrome/Safari) and OS (Windows 11/Android 14).
Why it's needed
It helps narrow down the device. If the log shows "iPhone 15" but the suspect only has a Nokia, you look for a hidden device.
Referer
Spelled with one 'r' in tech — shows which website the user came from before hitting the current page.
Server Header
Tells you what software the criminal's server is running — e.g. Apache or Nginx.

SSL/TLS Certificates
What it is
The "Green Padlock" (HTTPS) that encrypts data.
Why it's needed
To check if a site is "Validated." Scammers often use free certificates like Let's Encrypt.
Full Form
Secure Sockets Layer / Transport Layer Security.
Issuer
The company that gave the certificate. If "State Bank" site is issued by "FreeSSL-Now," it is a fake.
Validity Period
Certificates usually last 90 days or 1 year. Scam sites have very recent "Start Dates."

SSL/TLS Certificates
Subject Alternative Name (SAN)
Shows other websites owned by the same certificate holder — helping you find a "network" of scam sites.
General Location
Click the padlock icon in the browser address bar.
Investigation Use
If you find a certificate serial number, you can ask the issuer for the account details of the person who requested it.
Fingerprint
A unique code (SHA-256) for that certificate. You can search this code to find other identical fake sites.
Misconception
"HTTPS" does not mean a site is safe; it just means the connection is private. Even a thief can have a padlock on their door.

URL Shorteners
What it is
Services that turn long links into tiny ones — e.g. bit.ly/xxxx.
Why it's needed
Criminals use them to hide scam site names in SMS/WhatsApp messages.
De-masking
You can see the real destination by adding a + sign at the end of many shortened links — e.g. bit.ly/xxxx+.
Analytics
Most shorteners keep public stats on how many people clicked and from which country.
Investigation Use
You contact the service (like Bitly or TinyURL) to get the IP address of the person who created the link.

ICANN Protocols
Full Form
Internet Corporation for Assigned Names and Numbers.
What it is
The global "Government" of the internet that sets rules for domains.
Why it's needed
When a criminal uses a .com or .net domain, they must follow ICANN's "Registrant Agreement."
UDRP
Uniform Domain-Name Dispute-Resolution Policy — a process to take back a domain if someone is impersonating a brand like sbi-bank-login.com.
WHOIS Accuracy
ICANN rules state that if a user provides "False WHOIS data," the registrar must cancel the domain.

ICANN Protocols
Police Role
You can use ICANN's "Accuracy Complaint" to get a scam site taken down quickly if the owner's info is fake.

GNSO
The group within ICANN that makes rules for generic domains like .com.
Key Insight
ICANN doesn't remove "Content" (like a bad video) — they only handle "Domains" (the name).

Robots.txt & Sitemap
Robots.txt
A file that tells Google which parts of a website not to show in search results.
Sitemap
A "List of Contents" for the website meant for search engines.
Why it's needed

Botnet Tracking — "Like Farms"
What it is
A network of thousands of "zombie" social media accounts controlled by one person.
Why it's needed
To identify "Political Misinformation" or "Fake Trends" on Twitter/X.
The Pattern
If 5,000 accounts post the exact same spelling mistake at the exact same second, it's a botnet.
Like Farms
Physical rooms in countries like China, India, or Vietnam where thousands of phones are rigged to "Like" posts.
Investigation Use
Finding the "Operator" who is being paid to destroy someone's reputation online.
